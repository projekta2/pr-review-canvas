# Plantilla de Pull Request — Básica

> Copia esto en la descripción de tu PR. Elimina este bloque de cita y cualquier sección que genuinamente no aplique — pero piénsatelo dos veces antes de saltarte «Riesgos».

## Resumen

¿Qué hace este PR, en una o dos frases? Escríbelo para alguien que no tiene ningún contexto.

## Contexto

- Incidencia / ticket: [enlace]
- Por qué ahora: [una línea — qué ha motivado este cambio]

## Qué ha cambiado

- [Cambio principal 1]
- [Cambio principal 2]
- [Cualquier cosa incidental, mencionada expresamente para que no parezca que se está ocultando]

## Cómo probarlo

1. [Paso 1]
2. [Paso 2]
3. [Lo que deberías ver si funciona]

## Riesgos

¿Cuál es el peor caso si esto está mal? ¿Qué no has probado, y por qué es aceptable?

## Checklist (autor)

- [ ] Lo he ejecutado en local y hace lo que dice el resumen
- [ ] He añadido o actualizado las pruebas para el nuevo comportamiento
- [ ] He actualizado los docs/README si esto cambia cómo se usa el proyecto
- [ ] He eliminado los logs de depuración, el código comentado y los TODOs que no pretendía dejar
- [ ] Este PR está limitado a una sola cosa — los cambios no relacionados están en un PR separado
