# Pull Request Template — Basic

> Copy this into your PR description. Delete this quote block and any section that genuinely doesn't apply — but think twice before skipping "Risks."

## Summary

What does this PR do, in one or two sentences? Write it for someone who has zero context.

## Context

- Issue / ticket: [link]
- Why now: [one line — what prompted this change]

## What changed

- [Main change 1]
- [Main change 2]
- [Anything incidental, called out so it doesn't look hidden]

## How to test this

1. [Step 1]
2. [Step 2]
3. [What you should see if it works]

## Risks

What's the worst case if this is wrong? What did you *not* test, and why is that acceptable?

## Checklist (author)

- [ ] I ran this locally and it does what the summary says
- [ ] I added or updated tests for the new behavior
- [ ] I updated docs/README if this changes how the project is used
- [ ] I removed debug logs, commented-out code, and TODOs I didn't mean to leave in
- [ ] This PR is scoped to one thing — unrelated changes are in a separate PR
